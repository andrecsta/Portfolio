Fraud Detection in Real Estate Transactions in NYC

Using PCA and Tensorflow's Autoencoder

The aim of this project is to detect fraudulent real estate transactions using machine learning techniques. The dataset used is from NYC Open Data and involves extensive cleaning, feature engineering, and the creation of two fraud scores using PCA and autoencoder models.

This project begins with a thorough exploratory analysis, choosing the ones that are most relevant for our goal. Those are the summarized steps:

Part 1 - Data Cleaning

Step 1: Substituting Missing Zip Codes

Issue:

Identified null values in the some columns during exploratory analysis.

Approach:

  • Created functions to find the mode (most common) results for each variable and filled missing values using lookalike groups.
  • Grouped values by Borough and Block codes and applied the functions to fill missing ZIP codes.
  • Checked and adjusted for groups without a mode value by considering the frequency of neighboring groups.
Step 2: Substituting Missing Values in Numeric Columns

Issue:

Observed missing values and zeros in columns such as FULLVAL, AVLAND, AVTOT, BLDFRONT, BLDDEPTH, LTFRONT, LTDEPTH.

Approach:

  • Replaced zeros with NaN values.
  • Utilized median values within groups of Zip and BLDGCL codes to fill NaN values, considering groups with at least 5 registries.
  • Applied a similar strategy using combinations of Zip, TAXCLASS, B, and BLOCK codes for each numeric variable.

Part 2.2 - Feature Engineering

Step 1: Creation of Area Variables
  • Calculated two area variables (AREA1 and AREA2) based on combinations of LTFRONT, LTDEPTH, BLDFRONT, and BLDDEPTH.
  • Derived a third area variable (AREA3) by combining AREA2 with the 'STORIES' column.

Step 2: Creation of Index Variables

  • Generated indexes by dividing FULLVAL, AVLAND, and AVTOT by the newly created area variables.
  • Calculated mean indexes for groups based on ZIP, TAXCLASS, and B variables.

Part 2.3 - Building the Model (Score 1 using PCA)

Step 1: Data Standardization
  • Standardized the features created in Part 2.2 using PCA.
Step 2: Applying PCA Algorithm
  • Used PCA to reduce dimensionality, aiming to explain a significant portion of the variance with fewer variables.
  • Visualized the cumulative sum of explained variance.
  • Chose the number of components based on the cumulative explained variance plot.
  • Trained and transformed the model using the chosen number of components.
  • Squared the results to emphasize differences between data points.
  • Created a fraud score (Score 1) by summing the squared PCA values.

Part 2.4 - Building the Model (Score 2 using Autoencoder)

Step 1: Creating the Autoencoder Model

  • Used the 10 principal components from PCA as input for an autoencoder neural network.
  • Split the data into training and test sets.
  • Trained the autoencoder model to learn the patterns in the data, focusing on minimizing mean squared error.
  • Evaluated the model performance using loss curves.
Step 2: Evaluating the Model
  • Loaded the trained autoencoder model.
  • Calculated the difference between the input and output of the autoencoder.
  • Created a fraud score (Score 2) based on the squared differences.
Step 3: Integration and Final Score
  • Integrated the two scores using a weighted combination.
  • Ranked the scores and calculated a final rank based on the weighted combination.
  • Combined the final rank with the original dataset for further analysis.

Conclusion

The project demonstrated a systematic approach to data cleaning, feature engineering, and model building, resulting in two distinct fraud scores that were integrated to provide a comprehensive final rank for potential fraudulent real estate transactions. The step-by-step explanation highlights the thoughtful consideration of data issues and the implementation of advanced machine learning techniques for fraud detection.
